## 2. 신규 후보 커뮤니티 조사

### 네이트판 (pann.nate.com)
- **URL**: https://pann.nate.com/talk
- **콘텐츠**: 일상 고민, 연예, 외모/패션 이야기 (카테고리: 톡톡, 엔터톡, 10대 이야기 등)
- **수집 방법**: HTTP + cheerio
- **난이도**: Easy ⭐
- **로그인**: 불필요 (읽기는 비로그인 가능)
- **봇 차단**: 없음 (curl로 200 OK, 전체 HTML SSR 반환)
- **렌더링**: SSR (jQuery 기반 전통 웹)
- **일일 게시글**: ~500+ (활발한 커뮤니티)
- **니즈 발굴 가치**: ⭐⭐ (뷰티 전용 카테고리 없음, 일반 고민 위주)
- **추천**: **추천** — 크롤링 최고 난이도, 여성 사용자 비율 높음
- **비고**: 뷰티 전용 게시판 없이 "톡톡"에 뷰티 관련 글 산재. 키워드 필터링 필요. 카테고리 ID: c20001(일반), c20003(고민), c20008(연애) 등

### 화해 (hwahae.co.kr)
- **URL**: https://www.hwahae.co.kr
- **콘텐츠**: 화장품 성분 분석, 리뷰, 제품 정보
- **수집 방법**: Playwright (CSR)
- **난이도**: Hard ⭐⭐⭐
- **로그인**: 리뷰 열람 시 앱/로그인 필요 가능성 높음
- **봇 차단**: AWS WAF 감지 (`aws_waf_referrer` 코드 존재)
- **렌더링**: Next.js CSR (`__NEXT_DATA__`, `_buildManifest` 확인)
- **일일 게시글**: N/A (리뷰 플랫폼, 게시판 형태 아님)
- **니즈 발굴 가치**: ⭐⭐⭐ (뷰티 리뷰의 보고, 성분 고민/피부 트러블 정보 풍부)
- **추천**: **보류** — 데이터 가치 최상이나 기술적 난이도 높음 (CSR + WAF + 앱 우선 정책)
- **비고**: 앱 중심 서비스. 웹에서 리뷰 접근이 제한적. API 엔드포인트 404 반환. 향후 API 리버스엔지니어링 가능 시 재검토

## 5. 구현 우선순위 추천

### Phase 3 (난이도 높음, 선택적)

## 6. 기술 상세 노트

### 인코딩 주의사항
- **뽐뿌**: EUC-KR 인코딩 (`charset=euc-kr`)
- **네이트판**: UTF-8
- **DC**: UTF-8
- **속닥**: UTF-8 (Next.js)

### 봇 차단 수준
- **없음**: 네이트판, 뽐뿌, DC, 속닥, 글로우픽, 언니의파우치
- **Cloudflare Turnstile**: 에펨코리아 (HTTP 430)
- **AWS WAF**: 화해
- **플랫폼 보안**: 네이버/다음 카페 (로그인 필수)

### 기존 인프라 재활용
- `community_posts` 테이블에 `source_platform` 추가로 저장 가능
- 현재 지원: `naver_cafe`, `theqoo`, `instiz`
- 추가 필요: `socdoc`, `nate_pann`, `dcinside`, `glowpick`, `unpa`
