# 방법론 라우터 (Methodology Router)

> 작업 유형을 판단하고 최적의 방법론을 자동 선택한다.
> 모든 작업 시작 시 이 문서를 참조하여 방법론을 결정한다.

---

## 자동 판단 흐름

```
작업 요청 수신
  ↓
어떤 유형인가?
  │
  ├─ 분석/분류 작업 → MECE
  ├─ 소비자 이해 작업 → JTBD
  ├─ 반복 실행 작업 → OODA Loop
  ├─ 창작/기획 작업 → Double Diamond
  ├─ 리소스 배분 작업 → Pareto 80/20
  ├─ 복합 작업 → 2개 이상 조합
  └─ 단순 실행 → 방법론 불필요, 바로 실행
```

---

## 1. MECE — 분석/분류할 때

### 언제 사용
- 데이터를 카테고리로 나눌 때
- 니즈/댓글/포스트를 분류할 때
- "빠진 건 없나? 겹치는 건 없나?" 판단이 필요할 때

### 트리거 키워드
`분류해줘`, `카테고리`, `나눠줘`, `분석해줘`, `종류별로`

### 적용 방법

**Before (MECE 없이):**
```
댓글 분류:
- 추천 요청 (61건)
- 비교 요청 (18건)  ← "A vs B 뭐가 나아요?"는 추천인가 비교인가?
- 구매 의향 (34건)  ← "사야겠다"는 구매인가 추천 수용인가?
```

**After (MECE 적용):**
```
1단계: 상호 배타적(ME) 기준 정의
  - 추천 요청 = "뭐 써?" (제품 모름)
  - 비교 요청 = "A vs B" (제품 알고 선택 고민)
  - 구매 의향 = "이거 산다" (결정 완료)
  → 겹침 없음 ✓

2단계: 전체 포괄(CE) 확인
  - 위 3개에 안 들어가는 댓글 있나?
  - "이거 별로였어" → 불만 카테고리 추가 필요
  → 빠짐 없음 ✓
```

### 실전 체크리스트
1. [ ] 각 카테고리의 정의가 명확한가? (한 댓글이 두 카테고리에 동시에 들어가면 안 됨)
2. [ ] 분류 안 되는 항목이 있는가? (있으면 카테고리 추가)
3. [ ] "기타"가 전체의 10% 이하인가? (넘으면 카테고리 세분화 필요)

## 2. JTBD (Jobs-to-be-Done) — 소비자를 이해할 때

### 언제 사용
- YouTube 댓글에서 니즈를 추출할 때
- 포스트 소재를 기획할 때
- "이 사람이 진짜 원하는 게 뭐지?" 판단할 때
- 제품 매칭할 때

### 트리거 키워드
`니즈`, `왜 이걸 원하는지`, `소비자가`, `댓글 분석`, `소재 기획`

### 적용 방법

**Before (JTBD 없이):**
```
댓글: "쿠션 추천해주세요"
→ 니즈: 쿠션 추천 요청
→ 포스트: "쿠션 추천 TOP 5"
```

**After (JTBD 적용):**
```
댓글: "쿠션 추천해주세요"
→ JTBD 분석:
  - 상황(When): 여드름 피부인데 메이크업 해야 할 때
  - 동기(Want): 트러블 가리면서 피부 안 망치고 싶음
  - 결과(So that): 자신감 있게 외출하고 싶음
→ 진짜 Job: "트러블 커버 + 피부 보호를 동시에"
→ 포스트: "여드름 피부인데 쿠션 바르는 사람 나만 그래?"
  (제품이 아니라 상황에 공감 → 조회수 ↑)
```

### JTBD 문장 공식
```
[상황]일 때, [동기]하고 싶은데, [장벽] 때문에 [감정]
```

예시:
- "다이어트 중(상황)인데 피부가 안 말랐으면(동기) 좋겠는데, 뭘 해야할지 몰라서(장벽) 답답함(감정)"
- "선크림 사려는데(상황) 백탁 안 나는 걸(동기) 원하는데, 다 발라봐야 아는 거라(장벽) 짜증남(감정)"

### 실전 체크리스트
1. [ ] 표면 니즈(쿠션 추천)와 진짜 Job(트러블+커버)을 분리했는가?
2. [ ] 상황-동기-장벽-감정을 모두 채웠는가?
3. [ ] 포스트가 제품이 아니라 Job에 공감하는가?

## 3. OODA Loop — 매일 반복 실행할 때

### 언제 사용
- 일일 수집 → 분석 → 기획 → 게시 사이클
- 성과 추적 후 다음 행동 결정
- "어제 올린 게 잘 됐나? 오늘은 뭘 올리지?"

### 트리거 키워드
`매일`, `일일`, `루틴`, `성과 확인 후`, `다음에 뭘`

### 적용 방법

```
Observe (관찰):
  - YouTube 59채널 댓글 수집
  - Threads 성과 추적 (조회수, 참여율)
  - 커뮤니티 포스트 수집

Orient (방향 잡기) ← 여기가 핵심, 대부분 생략함:
  - 어제 포스트 성과 vs 평균 비교
  - "왜 잘 됐나 / 왜 안 됐나" 원인 분석
  - 새로 발견된 트렌드/니즈 확인
  - 6.3배 가설에 부합하는 소재 있나?

Decide (결정):
  - 오늘 올릴 소재 3~5개 선정
  - 패턴 A~E 중 선택
  - 주제 태그 결정

Act (실행):
  - 토론 시스템으로 포스트 작성
  - 게시
  - 다시 Observe로 → 루프
```

### Orient 단계 질문 리스트 (매일 체크)
1. 어제 포스트 조회수가 평균보다 높았나/낮았나?
2. 높았다면 어떤 패턴/소재였나? → 오늘도 비슷하게
3. 낮았다면 뭐가 달랐나? → 오늘은 바꿔서
4. 새로 수집된 데이터에서 강한 니즈가 있나?
5. X 트렌드에 탈 만한 게 있나?

## 4. Double Diamond — 창작/기획할 때

### 언제 사용
- 포스트 작성 (토론 시스템)
- 콘텐츠 기획
- 새로운 기능/시스템 설계

### 트리거 키워드
`만들어줘`, `기획해줘`, `작성해줘`, `설계해줘`, `포스트`

### 적용 방법 (토론 시스템에 내장)

```
◇ 1차 발산 — 가이드 에이전트
  - 패턴 5개 중 후보 2~3개
  - 훅 후보 5개
  - 본문 구조 후보 3개

◆ 1차 수렴 — 빈이 에이전트
  - 빈이 톤에 맞는 패턴 1개 선택
  - 훅 1개 선택 또는 수정

◇ 2차 발산 — 빈이 에이전트
  - 선택한 훅+구조로 초안 작성
  - 대안 표현 제시

◆ 2차 수렴 — 가이드 에이전트
  - 체크리스트 10개 검증
  - 통과 → 최종 확정
  - 미통과 → 구체적 수정 요청 (다시 2차 발산)
```

### 핵심 규칙
- **발산 단계에서 판단하지 않는다** (아이디어를 많이 내는 게 목적)
- **수렴 단계에서 새 아이디어를 내지 않는다** (선택하는 게 목적)
- 발산과 수렴을 섞으면 품질이 떨어짐

## 5. Pareto 80/20 — 리소스를 배분할 때

### 언제 사용
- 59개 채널 중 어디에 집중할지
- 시간이 부족할 때 우선순위 결정
- "다 할 수 없으니 뭘 먼저?"

### 트리거 키워드
`우선순위`, `집중`, `효율`, `시간 부족`, `뭘 먼저`

### 적용 방법

**채널 관리:**
```
59개 채널의 니즈 밀도 계산
→ 상위 20% (12개) = 전체 니즈의 80% 생산
→ 매일 수집: 상위 12개만
→ 주간 수집: 나머지 47개
→ API 쿼터 80% 절약
```

**포스트 소재:**
```
니즈 50개 발견
→ 상위 20% (10개) = 조회수 80% 차지할 소재
→ 이번 주 포스트: 상위 10개에서만 선택
→ 나머지는 다음 주 또는 폐기
```

**성과 분석:**
```
7개 포스트 중
→ 상위 2개 (목이버섯 15K, 도시락 8.7K) = 전체 조회수의 88%
→ 이 2개의 공통점 분석 → 다음 포스트에 적용
```

## 6. 방법론 조합 매트릭스

| 작업 | 주 방법론 | 보조 방법론 |
|------|----------|-----------|
| YouTube 댓글 니즈 분석 | **JTBD** (진짜 니즈 추출) | **MECE** (카테고리 분류) |
| 포스트 소재 기획 | **JTBD** (소비자 공감) | **Pareto** (상위 니즈 선택) |
| 포스트 작성 (토론) | **Double Diamond** (발산→수렴) | - |
| 일일 운영 사이클 | **OODA** (관찰→판단→실행) | **Pareto** (우선순위) |
| 채널/데이터 관리 | **Pareto** (상위 20% 집중) | **MECE** (카테고리 정리) |
| 성과 분석 | **OODA Orient** (왜?) | **Pareto** (상위 분석) |

## 7. AI 자동 판단 로직

작업 요청을 받으면 아래 순서로 판단:

```python
def select_methodology(task):
    keywords = extract_keywords(task)

    if any(k in keywords for k in ['분류', '카테고리', '나눠', '종류별']):
        return 'MECE'

    if any(k in keywords for k in ['니즈', '소비자', '왜', '진짜 원하는']):
        return 'JTBD'

    if any(k in keywords for k in ['매일', '루틴', '성과 후', '다음에']):
        return 'OODA'

    if any(k in keywords for k in ['만들어', '기획', '작성', '포스트']):
        return 'Double Diamond'

    if any(k in keywords for k in ['우선순위', '집중', '효율', '먼저']):
        return 'Pareto'

    # 복합 판단
    if '분석' in keywords and '기획' in keywords:
        return 'JTBD + Double Diamond'

    if '수집' in keywords and '분석' in keywords:
        return 'OODA + MECE'

    return None  # 방법론 불필요, 바로 실행
```

### 사용 예시

```
사용자: "YouTube 댓글에서 니즈를 분류해줘"
→ "니즈" → JTBD, "분류" → MECE
→ 조합: JTBD로 진짜 니즈 추출 + MECE로 카테고리 정리

사용자: "오늘 포스트 뭘 올리지?"
→ "오늘" → OODA, "포스트" → Double Diamond
→ 조합: OODA로 어제 성과 분석 → Double Diamond로 포스트 작성

사용자: "59개 채널 관리 어떻게?"
→ "관리" → Pareto
→ 상위 20% 집중 수집, 나머지 주간 수집
```
