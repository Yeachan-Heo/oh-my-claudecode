# YouTube 댓글 니즈 분석 에이전트 가이드

> 모델: **opus** | 입력: community_posts (youtube) | 출력: 니즈 리포트 + Threads 소재

## 목적

YouTube 뷰티 영상의 대본(body)+댓글(comments)을 분석하여 소비자 니즈를 발굴하고, Threads 포스트 소재를 생성한다.

**핵심 가설**: 여성 니즈 + 불편함 = 6.3배 전환율. 불만/고민 댓글에서 Threads 소재를 우선 생성한다.

---

## 분석 흐름

### Phase 1: 데이터 추출 (배치 처리)

1. DB에서 `analyzed=false`인 YouTube 포스트를 조회
2. 컨텍스트 크기 제한으로 **한 번에 최대 10개 포스트**씩 배치 처리
3. 각 포스트: `title` + `body`(대본) + `comments` 전체를 읽음

**DB 조회 (임시 스크립트)**:
```sql
-- 미분석 YouTube 포스트 조회
SELECT id, title, body, comments, source_cafe, like_count, comment_count, view_count
FROM community_posts
WHERE source_platform = 'youtube' AND analyzed = false
ORDER BY collected_at DESC;

-- 배치 조회 (10개씩)
SELECT id, title, body, comments, source_cafe, like_count, comment_count, view_count
FROM community_posts
WHERE source_platform = 'youtube' AND analyzed = false
ORDER BY collected_at DESC
LIMIT 10 OFFSET 0;  -- OFFSET을 10씩 증가

-- 키워드 필터 조회 (니즈 밀도 높은 것만)
SELECT id, title, body, comments, source_cafe, like_count, comment_count, view_count
FROM community_posts
WHERE source_platform = 'youtube' AND analyzed = false
  AND (
    comments::text LIKE '%추천%'
    OR comments::text LIKE '%고민%'
    OR comments::text LIKE '%건조%'
    OR comments::text LIKE '%트러블%'
    OR comments::text LIKE '%어디서 사%'
    OR comments::text LIKE '%대안%'
    OR comments::text LIKE '%비추%'
  )
ORDER BY comment_count DESC;
```

### Phase 2: 영상 단위 분석

각 영상에 대해 4단계 분석을 수행한다.

#### 1. 영상 카테고리 분류

제목 + 대본에서 영상 유형을 판별한다:

| 유형 | 판별 기준 | 니즈 밀도 |
|------|-----------|-----------|
| **리뷰** | 특정 제품명 반복, 장단점 언급 | 높음 — 댓글에서 대안/비교 요청 빈번 |
| **추천** | "추천", "BEST", "TOP" 등 제목 패턴 | 높음 — "나도 써봤는데" 댓글 多 |
| **비교** | "vs", "차이", "비교" 패턴 | 매우 높음 — 선택 고민 댓글 집중 |
| **루틴** | "모닝루틴", "나이트루틴", "스킨케어 루틴" | 중간 — 루틴 내 제품 질문 |
| **하울** | "하울", "언박싱", "구매" | 중간 — 가격/가성비 댓글 |
| **피부고민** | "트러블", "건조", "민감", "여드름" | 매우 높음 — 직접적 고민 토로 |
| **가성비** | "저렴", "가성비", "다이소", "만원 이하" | 높음 — 구매 의향 높은 댓글 |

#### 2. 대본 핵심 추출

유튜버가 대본에서 언급한 정보를 구조화한다:

```json
{
  "mentioned_products": [
    {
      "name": "아누아 어성초 토너",
      "sentiment": "positive",
      "claims": ["모공 축소", "진정 효과"],
      "price_mentioned": true
    }
  ],
  "recommendations": ["아누아 어성초 토너", "라운드랩 자작나무 수분크림"],
  "anti_recommendations": ["XX 선크림 — 백탁 심함"],
  "key_claims": ["건성에는 토너 2번 레이어링이 효과적"]
}
```

#### 3. 댓글 니즈 클러스터링

댓글을 **8개 카테고리**로 분류한다:

| 카테고리 | 설명 | 예시 댓글 | Threads 소재 우선도 |
|----------|------|-----------|---------------------|
| **제품 추천 요청** | 구체적 제품/카테고리 추천 요청 | "선크림 추천해주세요", "토너 뭐 써요?" | ★★★★★ |
| **제품 불만/실패** | 제품 사용 후 불만족 | "이거 써봤는데 안 맞아요", "돈 버렸어" | ★★★★★ |
| **피부 고민** | 본인 피부 상태 고민 토로 | "건조해서 미치겠어요", "트러블 안 나는 거 있나요" | ★★★★☆ |
| **가격/가성비** | 가격 비교, 가성비 탐색 | "이거 너무 비싸요", "저렴한 대안 없나요?" | ★★★★☆ |
| **구매 의향** | 구매 의사/계획 표현 | "이거 사야겠다", "장바구니 담았어요" | ★★★☆☆ |
| **비교 요청** | 두 제품 간 비교 | "A랑 B 중에 뭐가 나아요?" | ★★★★★ |
| **사용법 질문** | 사용 방법/순서 질문 | "이거 아침에 써도 되나요?", "순서가 어떻게?" | ★★★☆☆ |
| **부작용/알러지** | 부작용, 알러지 경험/우려 | "이거 쓰고 올라왔어요", "민감성도 괜찮나요?" | ★★★★☆ |

**분류 규칙**:
- 하나의 댓글이 여러 카테고리에 해당하면, 가장 강한 신호를 가진 카테고리 1개에 배정
- 단순 감탄("대박", "오") 또는 무의미 댓글은 분류하지 않음
- 광고/스팸 댓글은 제외

#### 4. 대본-댓글 교차 분석

유튜버 의견과 소비자 반응을 대조하여 니즈 시그널을 발견한다:

| 패턴 | 의미 | Threads 소재 활용 |
|------|------|-------------------|
| 유튜버 추천 ↔ 댓글 불만 | **강한 니즈 시그널** — 기대와 현실의 갭 | "~~ 추천 많이 받는데 진짜일까?" (문제공감형) |
| 유튜버 비추 ↔ 댓글 대안 요청 | **제품 공백 발견** — 시장에 빈자리 | "~~ 대신 쓸 수 있는 거 찾았다" (비교형) |
| 유튜버 미언급 ↔ 댓글 반복 질문 | **숨은 니즈** — 유튜버도 모르는 소비자 관심 | "아무도 안 알려주는 ~~ 꿀팁" (솔직후기형) |
| 유튜버 추천 ↔ 댓글 동의 | 검증된 제품 — 소재로 안전 | "다들 좋다고 하는 ~~ 써봤다" (솔직후기형) |
| 유튜버 추천 ↔ 댓글 가격 불만 | **가성비 니즈** — 대안 제품 기회 | "~~ 대신 반값에 쓸 수 있는 거" (비교형) |

### Phase 3: 니즈 집계 + Threads 소재 생성

1. **배치별 니즈를 누적 집계** — 동일 니즈가 여러 영상에서 발견되면 빈도 카운트 증가
2. **빈도 TOP 니즈 → Threads 포스트 소재로 변환**
3. **6.3배 가설 적용**: 불만/고민 카테고리(제품 불만, 피부 고민, 부작용) 댓글에서 발견된 니즈를 소재 생성 우선순위 최상위로 배치

**소재 생성 포맷 매핑**:

| 댓글 니즈 카테고리 | 추천 포맷 | 예시 훅 |
|-------------------|-----------|---------|
| 제품 불만/실패 | 문제공감형 | "~~ 쓰고 후회한 사람 나만 있어?" |
| 피부 고민 | 문제공감형 | "환절기만 되면 피부가 미쳐 날뛰는 사람?" |
| 비교 요청 | 비교형 | "A vs B 한 달 써본 솔직 비교" |
| 제품 추천 요청 | 입문추천형 | "처음 쓰기 좋은 ~~ TOP 3" |
| 가격/가성비 | 비교형 | "반값인데 효과 비슷한 ~~ 대체품" |
| 부작용/알러지 | 실수방지형 | "민감성이 절대 쓰면 안 되는 성분 3가지" |
| 구매 의향 | 솔직후기형 | "3주 써보고 솔직히 말한다" |
| 사용법 질문 | 솔직후기형 | "순서 하나 바꿨더니 피부가 달라졌다" |

### Phase 4: 콘텐츠 기획 (3~5개)

니즈 분석 결과에서 가장 강한 니즈 3~5개를 선정하고, 각각에 대해 Threads 포스트 기획안을 작성한다.

#### 선정 기준 (우선순위)

1. **여성니즈 + 불편함** 조합 우선 (6.3배 조회수 가설)
2. **댓글 빈도 높은 니즈** 우선
3. **대본-댓글 교차**에서 발견된 "유튜버 추천 but 소비자 불만" 우선

#### 각 기획안 포함 사항

| 항목 | 설명 |
|------|------|
| **제목/훅** | 첫 문장 — 스크롤 멈추게 하는 한 줄 |
| **포맷** | 문제공감형 / 솔직후기형 / 비교형 / 입문추천형 / 실수방지형 |
| **타겟 니즈** | 어떤 소비자 고민을 타겟하는지 |
| **근거 댓글** | 이 기획의 근거가 된 실제 댓글 2~3개 |
| **언급 제품** | 포스트에서 다룰 제품명 |
| **예상 반응** | 어떤 댓글/공감이 달릴지 예측 |
| **셀프댓글 초안** | 워밍업 이후 제휴링크용 댓글 (워밍업 기간에는 생략) |

#### 기획안 예시

```markdown
### 기획 1: "환절기만 되면 피부가 미쳐 날뛰는 사람?"

- **포맷**: 문제공감형
- **타겟 니즈**: 환절기 건조/트러블 (피부 고민, 빈도 45건)
- **근거 댓글**:
  - "건조해서 미치겠어요 수분크림 아무리 발라도 소용없음" (좋아요 23)
  - "환절기만 되면 트러블 올라와서 외출도 못 해요" (좋아요 18)
  - "크림 바꿨더니 오히려 더 건조해짐 ㅠ" (좋아요 12)
- **언급 제품**: 라운드랩 자작나무 수분크림, 아누아 어성초 토너
- **예상 반응**: "저도요ㅠㅠ", "이거 써봤는데 진짜 좋아요", "건성인데 뭐 써요?"
- **셀프댓글 초안**: "저는 요즘 이거 쓰고 있는데 확실히 달라요 → [쿠팡 링크]"
```

## 컨텍스트 초과 대응 전략

YouTube 영상은 대본(body)이 길고 댓글이 수백 개일 수 있어 컨텍스트 초과가 발생한다.

### 전략 1: 배치 분석 (기본)

- 10개 포스트씩 배치로 나눠 분석
- 각 배치 결과를 `data/youtube-needs-batch-N.md`에 저장
- 마지막 배치 후 모든 배치 결과를 종합하여 최종 리포트 생성

```
배치 1 (10개) → data/youtube-needs-batch-1.md
배치 2 (10개) → data/youtube-needs-batch-2.md
...
종합 → data/youtube-needs-YYYY-MM-DD.md
```

### 전략 2: 2단계 분석 (대량 데이터, 50개+)

1단계: 각 포스트를 1줄 요약으로 압축
```
[영상ID] 리뷰 | 아누아 토너 | 니즈: 건조개선, 가성비, 트러블진정
[영상ID] 비교 | 선크림 3종 | 니즈: 백탁없는선크림, 자외선차단, 순한성분
```
→ 100개 포스트도 ~100줄로 압축

2단계: 요약 목록에서 패턴/클러스터 분석

### 전략 3: 카테고리별 분석 (포커스 모드)

영상 유형별로 필터해서 분석:
```sql
-- 리뷰 영상만 (제목에 '리뷰' 포함)
SELECT * FROM community_posts
WHERE source_platform = 'youtube' AND analyzed = false
  AND title LIKE '%리뷰%'
ORDER BY comment_count DESC LIMIT 10;

-- 비교 영상만
SELECT * FROM community_posts
WHERE source_platform = 'youtube' AND analyzed = false
  AND (title LIKE '%vs%' OR title LIKE '%비교%' OR title LIKE '%차이%')
ORDER BY comment_count DESC LIMIT 10;
```
카테고리별로 니즈의 깊이를 확보할 수 있다.

### 전략 4: 키워드 기반 선별 (효율 모드)

DB에서 댓글에 특정 키워드가 포함된 포스트만 조회:
```sql
SELECT * FROM community_posts
WHERE source_platform = 'youtube' AND analyzed = false
  AND (
    comments::text LIKE '%추천%'
    OR comments::text LIKE '%고민%'
    OR comments::text LIKE '%건조%'
    OR comments::text LIKE '%대안%'
    OR comments::text LIKE '%비추%'
    OR comments::text LIKE '%어디서 사%'
  )
ORDER BY comment_count DESC;
```
니즈 밀도 높은 포스트만 분석하여 효율을 극대화한다.

## 실행 절차

### 1. 데이터 확인
```sql
-- 미분석 YouTube 포스트 수 확인
SELECT count(*) FROM community_posts
WHERE source_platform = 'youtube' AND analyzed = false;
```

### 2. 배치 분석 실행

각 배치(10개)에 대해:
1. DB에서 포스트 조회 (title, body, comments)
2. Phase 2 분석 수행 (카테고리 분류 → 대본 추출 → 댓글 클러스터링 → 교차 분석)
3. 배치 결과를 `data/youtube-needs-batch-N.md`에 저장

### 3. 종합 리포트 생성

모든 배치 완료 후:
1. 배치 파일들을 읽어서 니즈 집계
2. 빈도 TOP 니즈 추출
3. Threads 소재 생성 (6.3배 가설 적용)
4. `data/youtube-needs-YYYY-MM-DD.md`에 최종 리포트 저장

### 4. 콘텐츠 기획

종합 리포트에서 가장 강한 니즈 3~5개를 선정하여 기획안 작성:
1. Phase 4 선정 기준에 따라 니즈 3~5개 선정
2. 각 니즈에 대해 기획안 작성 (훅, 포맷, 근거 댓글, 제품, 예상 반응, 셀프댓글)
3. `data/content-plans-YYYY-MM-DD.md`에 저장

### 5. DB 업데이트
```sql
-- 분석 완료 마킹
UPDATE community_posts
SET analyzed = true,
    extracted_needs = '["카테고리_키워드1", "카테고리_키워드2"]'::jsonb
WHERE id IN ('분석한_포스트_id_목록');
```

## 출력물 형식

### 배치 결과 (`data/youtube-needs-batch-N.md`)

```markdown
# YouTube 니즈 분석 배치 N (YYYY-MM-DD)

## 분석 대상
- 포스트 수: 10
- 총 댓글 수: 523

## 영상별 분석
```

## DB 테이블 참조

분석 대상: `community_posts` 테이블

| 컬럼 | 용도 |
|------|------|
| `id` | 포스트 고유 ID |
| `source_platform` | `'youtube'`로 필터 |
| `source_cafe` | `youtube_{채널핸들}` |
| `title` | 영상 제목 |
| `body` | 영상 대본 (자막/설명) |
| `comments` | JSONB 배열 `[{nickname, text, like_count}]` |
| `like_count` | 영상 좋아요 수 |
| `comment_count` | 댓글 수 |
| `view_count` | 조회수 |
| `analyzed` | `false` → 미분석, `true` → 분석 완료 |
| `extracted_needs` | 분석 후 추출된 니즈 키워드 배열 |

## 주의사항

- **API 비용 $0**: Claude Code가 직접 분석하므로 Anthropic API 호출 없음
- **대본이 없는 영상**: `body`가 null/빈 문자열이면 댓글만으로 분석 (Phase 2의 2번/4번 스킵)
- **댓글이 적은 영상**: `comment_count < 5`이면 분석 가치 낮음 → 스킵 권장
- **중복 니즈 제거**: 동일 니즈가 다른 영상에서 발견되면 빈도를 합산하고 대표 댓글은 좋아요 수 높은 것 선택
- **임시 스크립트 규칙**: DB 조회 스크립트는 `_` 접두사 → 프로젝트 루트에서 실행 → 실행 후 삭제
