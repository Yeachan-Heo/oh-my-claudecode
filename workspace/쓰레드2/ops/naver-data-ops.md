## 2. 키워드 확장 규칙 (L1 → L2 → L3)

### 원칙
- **L1**: 카테고리 대표 키워드 (브로드 매칭, 월 검색량 10만+)
- **L2**: L1 파생 세부 키워드 (수식어 추가, 월 검색량 1만~10만)
- **L3**: L2 파생 롱테일 키워드 (특정 니즈/상황, 월 검색량 1천~1만)

### 확장 패턴

```
L1: "선크림"
  L2: "선크림 추천", "지성 선크림", "여름 선크림", "순한 선크림"
    L3: "지성 피부 선크림 추천", "땀에 강한 선크림", "화이트캐스트 없는 선크림"

L1: "영양제"
  L2: "영양제 추천", "면역 영양제", "피로회복 영양제", "아침 영양제"
    L3: "직장인 영양제 추천", "30대 여성 영양제", "피로회복 영양제 순위"

L1: "단백질바"
  L2: "단백질바 추천", "저칼로리 단백질바", "맛있는 단백질바"
    L3: "헬스 단백질바 순위", "포만감 있는 단백질바", "당 없는 단백질바"
```

### 확장 명령 (search.py 자동 확장)
```bash
# 자동 확장 (2라운드): L1 → L2 자동 추출
python3 /home/sihun92/projects/oh-my-claudecode/workspace/naver-keyword-search/search.py "선크림"

# 확장 없이 L1만 (--no-expand)
python3 /home/sihun92/projects/oh-my-claudecode/workspace/naver-keyword-search/search.py "선크림" --no-expand
```

## 3. 검색량 × 트렌드 매트릭스

### 등급 기준

| 등급 | 월 검색량 | 트렌드 방향 | 설명 |
|------|-----------|-------------|------|
| **A+** | 10만+ | 상승 (↑) | 최우선 기획 — 즉시 콘텐츠화 |
| **A** | 10만+ | 유지 (→) | 우선 기획 — 안정적 수요 |
| **B** | 1만~10만 | 상승 (↑) | 틈새 공략 — 트렌드 선점 |
| **C** | 1만~10만 | 유지 (→) | 보조 기획 — 다른 신호와 조합 |
| **D** | 1만 미만 | 하락 (↓) | 보류 — 수요 감소 중 |

### 트렌드 방향 판단 (30일 기준)
- **상승 (↑)**: 최근 7일 평균 > 이전 23일 평균 × 1.1
- **유지 (→)**: ±10% 이내
- **하락 (↓)**: 최근 7일 평균 < 이전 23일 평균 × 0.9

### 매트릭스 예시

```
뷰티 카테고리 현황:
  선크림    월 82만 검색 + 상승 → A+ (여름 시즌 진입)
  세럼      월 21만 검색 + 유지 → A  (안정적 수요)
  올리브영  월 48만 검색 + 상승 → A+ (신제품/이벤트)
  클렌징    월 15만 검색 + 유지 → A
  토너      월 9만 검색  + 하락 → C  (세럼 전환 트렌드)
```

## 4. 실행 절차

### 4-1. 수집 명령 (`/수집 네이버`)
```bash
# 카테고리별 L1 검색량 수집
python3 /home/sihun92/projects/oh-my-claudecode/workspace/naver-keyword-search/search.py \
  "선크림" "클렌징" "토너" "세럼" "올리브영" \
  "영양제" "프로바이오틱스" "비타민" "오메가3" \
  "생활용품" "방향제" "세제" "수납" \
  "다이어트 식품" "단백질바" "곤약" "닭가슴살" \
  --no-expand

# 30일 트렌드 수집 (카테고리당 대표 키워드 5개 이내)
python3 /home/sihun92/projects/oh-my-claudecode/workspace/naver-keyword-search/trend.py \
  "선크림" "세럼" "올리브영" "영양제" "단백질바" --period 30
```

### 4-2. 결과 저장
- `data/naver-keywords-{YYYY-MM-DD}.json` — 검색량 JSON
- `data/naver-trends-{YYYY-MM-DD}.json` — 트렌드 JSON

### 4-3. 분석 → 기획 연계

1. 검색량 + 트렌드로 등급 산출 (매트릭스 적용)
2. **A+/A 등급 키워드** → `/기획` 스킬에 우선 신호로 투입
3. **B 등급 상승 키워드** → 트렌드 선점 기획 (선제적 콘텐츠)
4. **C/D 등급** → 다른 소스(X트렌드, 벤치마크)와 교차 확인 후 결정

---

## 5. 에디터 참조 방법

`/기획` 또는 `/threads-plan` 실행 시:
- `data/naver-keywords-{오늘날짜}.json` 존재하면 자동 참조
- 파일 없으면 수집 먼저 실행: `/수집 네이버`
- 키워드 등급은 **A+ → A → B** 우선순위로 기획서 신호에 반영
