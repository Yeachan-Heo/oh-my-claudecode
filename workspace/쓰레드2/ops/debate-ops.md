# 포스트 토론 운영 절차 (Debate Ops)

> 이 파일은 `src/agents/post-debate-system.md`에서 분리된 **토론 절차**다.
> Threads 포스트 작성 시 반드시 이 시스템을 사용한다. (hard rule)
> 단독 작성 금지 — 두 에이전트의 토론을 거쳐야 포스트가 완성된다.

---

## 개요

| 에이전트 | 역할 | 참조 문서 |
|---------|------|----------|
| **가이드 에이전트** | 데이터 기반 품질 관리 | `ops/writing-guide-ops.md` |
| **빈이 에이전트** | 톤/감성/캐릭터 일관성 | `souls/bini-persona.md` |

```
소재 입력
  ↓
가이드 에이전트: 패턴 선택 + 구조 제안
  ↓
빈이 에이전트: 톤 변환 + 초안 작성
  ↓
가이드 에이전트: 체크리스트 검증 + 피드백
  ↓
빈이 에이전트: 수정 반영
  ↓
가이드 에이전트: 최종 승인
  ↓
사용자에게 전달
```

## 체크리스트 (가이드 에이전트 사용)

### 킬러 게이트 (K1~K4) — 이것부터 먼저

여기서 탈락하면 아래 볼 필요 없음.

| # | 체크 | 실패 예시 | 통과 예시 |
|---|------|----------|----------|
| K1 | "그래서 뭐?" 테스트 — 저장/공유할 구체적 정보가 있는가? | "피부 고민 있는 사람 공감해" | "철분 시금치 10배, 5분컷" |
| K2 | 이미지 필수 — 텍스트 온리 금지. 최소 1장 | (텍스트만) | (음식 사진 2장) |
| K3 | 채널 온브랜드 — 뷰티/건강/생활/다이어트 범위 내인가? | BTS 공연 교통 정보 | 선크림 비교 |
| K4 | 글자수 100~150자 타겟 (최대 200자, 500자 절대 금지) | 280자 에세이 | 115자 |

### 기본 체크 (10항목)

1. 첫 문장 20자 이내 (30자는 너무 김)
2. 첫 문장에 구체적 팩트/숫자/반전 포함
3. AI 말투 없음 ("~합니다", "여러분", "추천드립니다", "효과적" 금지)
4. 줄바꿈 충분 (한 문단 2줄 이내)
5. 이모지 2개 이하
6. 구어체 (ㅋㅋ, ㅜ, ~거든, ~임)
7. 워밍업이면 광고/제품링크 없음
8. 같은 포맷 연속 3회 아님
9. CTA가 구체적인가? ("댓글 좀" ❌ → "먹어봐"/"댓글로 적어줘" ✅)
10. 조회수 터지는 소재 유형 7개 중 하나에 해당

## 토론 라운드 (최소 2라운드, 최대 4라운드)

### Round 0: 킬러 게이트 (가이드 단독, 빈이 전달 전)

```
가이드가 소재를 받으면 먼저 K1~K4 판단:
  - K1: 이 소재에서 구체적 정보(숫자/제품명/팁)를 뽑을 수 있는가?
  - K2: 이미지를 붙일 수 있는가? (어떤 이미지?)
  - K3: 뷰티/건강/생활/다이어트 범위 내인가?
  - K4: 117자 내외로 완성할 수 있는가?

→ K1~K4 전부 YES → Round 1 진행
→ 하나라도 NO → 소재 교체 요청 (사유 명시)
```

### Round 1: 구조 제안

```
가이드 → 빈이:
  - 추천 패턴: A~E 중 선택 + 이유
  - 훅 후보 3개 (각 20자 이내, 팩트/숫자 포함)
  - 본문 구조 제안 (목표 글자수: 100~150자)
  - 이미지 제안 (어떤 사진/이미지를 붙일지)
```

### Round 2: 초안 작성 + 검증

```
빈이 → 가이드:
  - 훅 선택 (또는 수정)
  - 전체 초안

가이드 → 빈이:
  - 체크리스트 결과 (통과/실패 항목)
  - 실패 시 구체적 수정 요청
```

### Round 3 (필요 시): 수정 + 재검증

```
빈이 → 가이드:
  - 수정본

가이드:
  - 재검증 → 통과면 "승인"
  - 또 실패면 Round 4
```

### Round 4 (최대): 최종 합의

```
4라운드까지 합의 안 되면:
  가이드가 직접 수정안 제시 → 빈이가 톤만 조정 → 강제 완료
```

## 출력 형식

토론 완료 후 리드에게 전달하는 최종 보고:

```
== 포스트 토론 완료 ==

패턴: [A~E 중 선택된 것]
라운드: [몇 라운드 걸렸는지]
킬러 게이트: K1~K4 통과
체크리스트: 10/10 통과
글자수: [N자] (목표 100~150자)

[최종 포스트 본문]

이미지: [어떤 이미지를 붙일지 설명]
주제: [Threads 주제 태그]
워밍업 모드: O/X
정보 밀도: [포함된 구체적 정보]
```

## 팀 없이 단독 실행 시

```
1. 리드가 ops/writing-guide-ops.md 읽음 (가이드 역할)
2. 리드가 souls/bini-persona.md 읽음 (빈이 역할)
3. 리드 내부에서 두 관점을 번갈아 적용
4. 체크리스트 10개 검증
5. 최종본 출력
```

**팀(2 에이전트) 방식이 품질이 더 높으므로 가능하면 팀으로 실행.**
