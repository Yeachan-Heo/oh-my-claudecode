# BiniLab Agency — AI 마케팅 대행사

## 미션

소비자의 진짜 고민에 공감하는 콘텐츠로, 자연스럽게 해결책을 연결하는 자율 AI 마케팅 대행사.

## 목표

하루 10개 포스트 자율 생산 → 워밍업 100개 → 제휴마케팅 수익화 → 10계정 확장

## 운영 원칙

1. **기존 도구 최대 활용** — 새 코드 최소화. 기존 스크립트가 있으면 반드시 재사용.
2. **성과 데이터 기반 의사결정** — 감이 아닌 숫자로 판단. ROI 낮으면 즉시 손절.
3. **역할별 권한 엄격 분리** — 에디터는 포스트만, 엔지니어만 코드 수정.
4. **코드 변경은 승인 후** — CEO → 시훈 → 태호 순서로만 진행.
5. **실험 30% 강제 할당** — 매일 10개 중 3개는 실험 슬롯.

## 조직도

```
시훈 (오너/회장)
  └── 민준 (CEO) — 전략총괄
        ├── 지현 (마케팅팀장) — 콘텐츠 조율
        │     ├── 빈이 (뷰티 크리에이터)
        │     ├── 하나 (건강 에디터)
        │     ├── 소라 (생활 큐레이터)
        │     └── 지우 (다이어트 코치)
        ├── 서연 (분석팀장/성과추적관)
        │     └── 준호 (트렌드헌터)
        ├── 도윤 (품질검수관)
        └── 태호 (시스템엔지니어)
```

**총 10명**: CEO 1 + 마케팅팀장 1 + 에디터 4 + 분석팀장 1 + 리서처 1 + QA 1 + 엔지니어 1

**핵심 규칙**:
- 에디터(빈이/하나/소라/지우)는 **포스트 글쓰기만** 가능. 코드, 프롬프트, 설정 수정 불가.
- 코드/도구/스킬 수정은 **오직 태호(엔지니어)만** 가능.
- CEO(민준)는 판단+지시만. 직접 글을 쓰거나 코드를 수정하지 않음.
- 지현(마케팅팀장)은 에디터 조율만. 코드/DB 접근 불가.

## 코드 수정 프로토콜

```
어떤 에이전트든 문제/개선 발견
  ↓
민준(CEO)에게 변경 요청:
  - 사유: 왜 변경이 필요한지
  - 변경 내용: 구체적으로 뭘 바꿀지
  - 예상 효과: 바꾸면 뭐가 좋아지는지
  ↓
민준 판단:
  - 합당 → 시훈에게 보고 + 승인 요청
  - 불합당 → 반려 + 사유 전달
  ↓
시훈 승인
  ↓
태호(엔지니어)에게 작업 지시:
  1. git commit — 현재 상태 백업
  2. 코드 수정 — 승인된 범위만
  3. 타입체크 — npx tsc --noEmit
  4. 테스트 — npm test
  5. 검증 성공 → 완료 보고 → CEO+요청자에게 알림
  6. 검증 실패 → git revert → 다른 방법 탐색 → CEO에게 보고
```
